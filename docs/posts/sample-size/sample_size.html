Sample Size for A/B Hypothesis Testing

Author

chris

Published

March 31, 2024

I had a conversation at work with a colleague about evaluation metrics and if small model improvements were actually statistically significant. After some research we concluded that sampling wasn’t an effective strategy, given the small amount of data, but it reminded me of A/B testing. A few years back I had derived a sample size calculator but didn’t fully document it so that’s what I intend to do here. Before I dive into the sample size derivation, I’ll briefly go over hypothesis testing in the A/B framework.

Typically, an A/B test is performed to evaluate the effectiveness of a treatment. The treatment could be a change to website, a change to a process, etc, and the metric used to compare the control and treatment groups might be sign-up rate, denial rate, etc. For this hypothetical example I’ll be using click-through rate to evaluate a change to a website.

Before performing an A/B test, there are a few parameters which are either already known or defined. You should already have the current click-through rate. And prior to running the test, the type I and II error rates need to be defined as well as minimum detectable change. Type I error is the false positive rate (i.e. accept there was a change due to the treatment when there wasn’t) and type II error is the false negative rate (i.e. reject there was a change due to the treatment when there was). So we will assume that these parameters are known. What isn’t known is the sample size of the test and the critical value where the null hypothesis is accepted or rejected…

why sample size is important

… ok now onto some more formal definitions. The number of clicks would be a binomial random variable which we can define as

\[ X \sim B(n, p) \]

where \(n\) is number of trials and \(p\) is the probability of a success. By definition, the expected value and variance for this random variable are

\[ E[X] = np, \ Var(X) = np(1 - p) \]

but we are more interested in the rate of clicks so we can define a new variable

\[ Y = \frac{X}{n} \]

and by properties of expectation and variance

\[ E[Y] = p, \ Var(Y) = \frac{p(1 - p)}{n} \]

so we have a formal mathematical definition of the distribution for click-through rate. Now onto hypothesis testing.

Let’s further define the current conversion rate as \(Y_c\) and the treatment group rate as \(Y_e\). For convenience we can define a new variable that is the difference between the two \(Y_d = Y_e - Y_c\). In our test, let’s say the null hypothesis is that the expected value of this difference is zero:

\[ H_0: E[Y_d] = 0 \]

and the alternative hypothesis that the difference is equal to a value \(d\) (i.e. the treatment had a positive effect):

\[ H_a: E[Y_d] = d \]

in practice, the alternative hypothesis would be \(\ge\) or \(\le\) instead of \(=\) but I’m ignoring that for now. At this point, it’s also important to note that there is a difference between test types; one sided or two sided. One sided only evaluates for a change in one direction whereas two sided tests in both directions (i.e. could be less than \(d\) or greater than \(d\)). This is important because it affects the critical value and thus sample size (elaborate here or later…).

let’s show graphically what the test statistic distribution looks like under the null and alternative hypothesis along with type I and type II errors as well as the critical value \(x_d\). note that when the number of binomial samples is large enough, we can approximate it with a normal distribution. this means that we can use Z-scores and assume the distribution is Gaussian.

library(ggplot2)

mu_null <- 0
n <- 100000
alpha <- 0.05

z_alpha <- qnorm(1 - alpha)

density_null <- density(rnorm(n, mu_null, 1))
df_null <- data.frame(x = density_null$x, y = density_null$y)

ggplot() +
  geom_ribbon(data = df_null[df_null$x < z_alpha, ],
              aes(x = x, ymax = y, fill = "no change (true negative)"),
              ymin = 0, alpha = 0.3) +
  geom_ribbon(data = df_null[df_null$x >= z_alpha, ],
              aes(x = x, ymax = y, fill = "type I error (false positive)"),
              ymin = 0, alpha = 0.3) +
  scale_fill_manual(values = c("black", "red")) +
  geom_segment(aes(x = mu_null, y = 0, xend = z_alpha, yend = 0)) +
  geom_text(aes(0.75, 0, label = "d_alpha", vjust = -1)) +
  geom_vline(xintercept = 0, linetype="dashed", linewidth = 0.3) +
  geom_vline(xintercept = z_alpha, linetype="dashed", linewidth = 0.3) +
  geom_text(aes(x = 1.9, label = "\nx_d", y = 0, vjust = 0.75), angle=0) +
  scale_x_continuous(limits = c(-3, 3)) +
  theme(axis.text.x=element_blank(), axis.ticks.x=element_blank(),
        axis.text.y=element_blank(), axis.ticks.y=element_blank(), axis.title.y=element_blank(),
        legend.title=element_blank()) +
   xlab(mu_null)

note that above I’ve also defined the distance from the mean under the null to the critical value as \(d_\alpha\). now let’s look at the alternative hypothesis when \(d = 0.4\)

the distance from the mean under the alternative hypothesis to the critical value is defined as \(d_\beta\). you can see that these two distances add to the difference in means between null and alternative hypothesis. by definition, the minimum detectable change (difference between the mean under the null and the mean under the alternative hypothesis) is equal to \(d\), so we can also define the following

\[d = d_\alpha + d_\beta\]

in other words, the critical value allows us to relate minimum detectable effect with type I and type II errors…

in order to quantify \(d_\alpha\) and \(d_\beta\), we need the expected value and variance of \(Y_d\) under the null and alternative hypothesis. the expected value is pretty straighforward:

\[ E[Y_d] = E[Y_e] - E[Y_c] = p_e - p_c \]

whereas the variance is (assuming equal number of samples):

\[ Var(Y_d) = Var(Y_e) + Var(Y_c) = \frac{p_e(1 - p_e)}{n} + \frac{p_c(1 - p_c)}{n} \]

so under the null hypothesis where \(p_e = p_c\) we have:

\[ H_0: E[Y_d] = 0, \ Var(Y_d) = \frac{2p_c(1 - p_c)}{n} \]

and under the alternative hypothesis:

\[ H_a: E[Y_d] = d, \ Var(Y_d) = \frac{p_c(1 - p_c) + p_e(1 - p_e)}{n} \]

using the normal approximation as previously noted, the distances are

\[ d_\alpha = Z_{1-\alpha} \sqrt{\frac{2p_c(1-p_c)}{n}} \]

\[ d_\beta = Z_{1-\beta}\sqrt{\frac{p_c(1 - p_c) + p_e(1 - p_e)}{n}} \]

with some rearranging

\[ d_\alpha = \frac{1}{\sqrt{n}} Z_{1-\alpha} \sqrt{2p_c(1-p_c)} \]

\[ d_\beta = \frac{1}{\sqrt{n}} Z_{1-\beta} \sqrt{p_c(1-p_c) + p_e(1-p_e)} \]

and so

\[ d = \frac{1}{\sqrt{n}} \biggl( Z_{1-\alpha} \sqrt{2p_c(1-p_c)} + Z_{1-\beta} \sqrt{p_c(1-p_c) + p_e(1-p_e)} \biggr) \]

solving for \(n\), the sample size of the test is

\[ n = \biggl( \frac{Z_{1-\alpha} \sqrt{2p_c(1-p_c)} + Z_{1-\beta} \sqrt{p_c(1-p_c) + p_e(1-p_e)}}{d} \biggr) ^2 \]

and the critical value to accept/reject the null

\[ x_d = p_c + d_\alpha = p_c + Z_{1-\alpha} \sqrt{\frac{2p_c(1-p_c)}{n}} \]

… this sample size accounts for the scenario where the change in rate is positive and the test is one-sided. For a two-sided test, the \(\alpha\) value for the Z-score would have to divided by two in order to account for false positives in both directions. additionally, the alternative hypothesis variance could take on two different values which results in two different sample size calculations. for example, the alternative conversion rate \(p_e\) could be either \(p_c + d\) or \(p_c - d\). which do we choose and does it make a difference in terms of sample size? let’s look at a plot of variance of as a function of \(p_e\) (recall this is \(E[Y_e]\)).

# plot of variance for alternative hypothesis as a function E(Y_e)

mu_e = seq(0.01, 0.99, by=0.01)  # i.e. p_e

var_e_fn <- function(p_e) {
  return(p_e * (1 - p_e) / n)
}

var_e = sapply(mu_e, var_e_fn)

ggplot(data = data.frame(x = mu_e, y = var_e)) +
  geom_point(aes(x = x, y = y), alpha = 0.5) +
  xlab("E(Y_e)") + ylab("Var(Y_e)")

so if we have a two-sided test, in order to be conservative, we should choose the value of \(p_e\) that is closest to 0.5 and maximizes the variance resulting in a larger sample size.
